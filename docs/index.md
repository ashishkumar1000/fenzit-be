# Project Documentation Index — fenzit-be (Jobzo Backend)

This is the **primary entry point** for AI-assisted development on the Jobzo
backend. Read this first.

## Project Overview

- **Type:** Monolith (single NestJS app + co-located Supabase migrations)
- **Primary Language:** TypeScript 5.7 (NodeNext ESM)
- **Architecture:** Modular monolith with JWT-scoped per-request data access
- **Runtime:** Bun ≥ 1.3.13
- **Database:** Supabase Postgres with RLS on every table
- **Object storage:** Cloudflare R2 (presigned PUT)
- **Auth:** Phone OTP → custom JWT (`@nestjs/jwt`, 7d expiry, signed with `SUPABASE_JWT_SECRET`)

### Quick Reference

- **Tech Stack:** NestJS v11 + Fastify v5 + Supabase (`@supabase/supabase-js` v2.108) + Cloudflare R2 (S3 SDK)
- **Entry Point:** `src/main.ts` → `AppModule` (`src/app.module.ts`)
- **API Base Path:** `/api/v1` (excludes `/health` and `/internal/webhooks/storage`)
- **Swagger (non-prod):** `/api/docs`
- **Architecture Pattern:** Modular monolith with per-request JWT-scoped Supabase client factory

## Getting Started

```bash
# Install dependencies
bun install

# Set up environment
cp .env.example .env  # then fill in real values

# Apply database migrations (via Supabase MCP — see project-context.md)

# Start dev server
bun run start:dev

# Run tests
bun run test           # unit
bun run test:e2e       # e2e + integration

# Run the RLS isolation test (HARD LAUNCH BLOCKER for any RLS-touching change)
bun run test:e2e -- rls-isolation
```
